⚠️ Common Confusion Points When Pipeline Starts

  • Topic injection vs discovery: If --topic-file is passed (from monitor_cloud.py), Discovery + Weighting are SKIPPED. The injected topic goes straight to Scripting. This is how cron jobs work — monitor picks the topic, pipeline produces it.
  • Topic retry loop: Pipeline tries up to 5 topics (MAX_TOPIC_RETRIES). If Scripting/Assembly fails on topic 1, it moves to topic 2. Forensic audit failure = HARD HALT (no retry).
  • Integration gates vs task agents: 13 integration gates sit BETWEEN task agents. They validate state handoffs. If a gate fails (missing/empty state key), the topic is marked failed and next topic is tried.
  • Upload is DISABLED: ResilientUploaderAgent saves to Google Drive, NOT YouTube. YouTube upload only happens via upload_approved.py after manual Telegram approval.
  • Post-pipeline agents run AFTER upload: They don't block the pipeline. Analytics, competitor intel, blog generation all happen after the main flow completes.
  • Workspace cleanup runs TWICE: Once in CleanupAgent (pre-pipeline) and once inline at start of execute_pipeline(). This is intentional — belt and suspenders.
  • Shorts are produced by ShortsOptimizationAgent (post) NOT the main pipeline: Main pipeline produces 1 main video. Shorts count is decided by publish_decision_engine.py (min 2). ShortsOptimizer runs post-pipeline to refine titles/CTAs.
  • Approval gate is separate from pipeline: Pipeline → approval_queue.json → Telegram → human approves → callback poller → upload_approved.py. The pipeline itself NEVER uploads to YouTube.
